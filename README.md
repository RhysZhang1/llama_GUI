# Llama GUI v2

一个基于纯 C + Win32 API 编写的 [llama.cpp](https://github.com/ggml-org/llama.cpp) 图形化启动与转换工具。
由AI辅助编写
无需手动记忆和输入冗长的命令行参数，通过简洁的窗口界面即可配置并运行 llama-cli / llama-server，以及转换 HuggingFace 模型为 GGUF 格式。

## 界面说明

### 主界面

程序启动后显示三个功能入口：

| 按钮 | 功能 |
|------|------|
| **🚀 运行** | 打开运行配置窗口，设置参数并启动 llama-cli 或 llama-server |
| **🔄 转化** | 打开模型转换窗口，将 HF 模型或 LoRA 转换为 GGUF 格式 |
| **📋 模型管理** | 管理模型列表（添加、编辑、删除），支持文件浏览选择 |

### 运行窗口

#### 运行模式

- **命令行模式** — 启动 `llama-cli`，在 CMD 窗口中交互对话
- **网页模式** — 启动 `llama-server`，自动打开浏览器访问 WebUI

#### 基本参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| ctx-size | 上下文窗口大小 | 8192 |
| ngl | GPU 加速层数 | 48 |
| 温度 (temp) | 生成随机性，越高越有创意 | 0.7 |
| 重复惩罚 | 抑制重复内容的强度 | 1.1 |
| 最大生成 (-n) | 单次最大生成 token 数 | 2048 |
| 端口 | 网页模式的监听端口 | 8081 |

#### 选项

| 选项 | 命令行参数 | 说明 |
|------|-----------|------|
| 多行输入模式 | `-mli` | 按两次回车发送消息 |
| MTP 推测解码 | `--spec-type draft-mtp --spec-draft-n-max 4` | 启用多头推测解码加速生成 |
| llama.cpp 路径 | 指定 `llama-cli` / `llama-server` 的完整路径 | 未添加到 PATH 时无需手动配置环境变量 |
| 启用额外参数 | 自定义 | 手动输入任意其他命令行参数 |

#### 命令预览

点击「📋 预览命令」后：
1. 根据当前参数自动生成完整命令
2. 命令显示在可编辑文本框中，**支持自由修改**
3. 检查无误后点击「▶ 确认运行」执行
4. 执行后程序自动退出

> 点击「↺ 重新生成」可根据参数重新生成命令（覆盖手动修改）。

### 转换窗口

#### 路径设置

所有路径输入框旁都有 **「浏览...」按钮**，无需手动输入路径。

| 字段 | 说明 | 浏览方式 |
|------|------|----------|
| llama.cpp 路径 | llama.cpp 仓库的本地文件夹 | 文件夹选择 |
| 模型源文件路径 | 要转换的 HF 格式模型文件夹 | 文件夹选择 |
| LoRA 源文件路径 | 要转换的 LoRA 适配器文件夹（可留空） | 文件夹选择 |
| 输出文件路径 | 转换后的 .gguf 文件保存位置 | 文件保存对话框 |
| --outtype | 输出量化格式 | 手动输入（如 f16, q8_0, q4_K_M） |

#### 转换类型

| 按钮 | 执行的脚本 | 说明 |
|------|-----------|------|
| 🔄 转换模型 | `convert_hf_to_gguf.py` | 将 HF 模型转为 GGUF |
| 🔧 转换 LoRA | `convert_lora_to_gguf.py` | 将 LoRA 适配器转为 GGUF |

> 点击后会弹出确认对话框展示完整命令参数，确认后在新的 CMD 窗口中执行。

### 模型管理

提供模型列表的**图形化管理**，无需手动编辑文本文件：

- **添加** — 输入名称 → 浏览选择 .gguf 文件 → 点击「添加」
- **编辑** — 左侧列表选择模型 → 修改名称/路径 → 点击「更新」
- **删除** — 选择模型 → 点击「删除」→ 确认
- **返回** — 回到主界面

> 所有操作**立即写入配置文件**（`llama_models.txt`），无需手动保存。

## 编译与安装

### 下载

从 [Release 页面](https://github.com/RhysZhang1/llama_GUI/releases) 下载最新版 `llama_gui.exe`，或克隆仓库自行编译。

### 编译要求

- Windows 7 或更高版本
- [MinGW-w64](https://www.mingw-w64.org/) 或 [MSYS2](https://www.msys2.org/) GCC 编译器

### 编译命令

```bash
gcc -o llama_gui.exe main.c run_window.c convert_window.c \
    -mwindows -s -O2 -lole32 -lshell32 -lcomctl32
```

> 编译后只需 `llama_gui.exe` 一个文件即可运行。首次运行会自动在同目录下生成配置文件。

### 运行

双击 `llama_gui.exe` 即可启动。配置文件将保存在 exe 所在目录。

## 配置文件说明

程序使用三个配置文件，均以 **UTF-8 编码**保存在 exe 同目录下。

### llama_config.txt — 运行配置

```
命令行                    ← 运行模式 (命令行/网页)
qwen3.6_27B_un           ← 上次选择的模型名称
65536                    ← ctx-size
48                       ← ngl
0.7                      ← 温度
1.1                      ← 重复惩罚
2048                     ← -n (最大生成长度)
8081                     ← 端口
1                        ← MLI 开关 (0=关 1=开)
0                        ← 额外参数开关 (0=关 1=开)
--top-k 40 --top-p 0.9   ← 额外参数字符串
1                        ← MTP 开关 (0=关 1=开)
1                        ← llama.cpp 路径开关 (0=关 1=开)
C:\llama.cpp\build\bin   ← llama.cpp 可执行文件所在文件夹路径
```

### llama_models.txt — 模型列表

```
3                        ← 模型数量
qwen3.6_27B_un           ← 模型1 名称
C:/models/qwen.gguf      ← 模型1 路径
gemma4_E4B               ← 模型2 名称
C:/models/gemma.gguf     ← 模型2 路径
deepseek_r1              ← 模型3 名称
D:/ai/deepseek.gguf      ← 模型3 路径
```

### llama_convert_config.txt — 转换路径配置

```
# outtype 参数
f16
# llama.cpp 路径 (文件夹)
C:\llama.cpp
# 模型源文件路径 (HF 格式文件夹)
D:\models\hf_model
# 转换后模型存储路径
D:\models\output.gguf
# LoRA 源文件路径 (可为空)
```

> 以 `#` 开头的行是注释，可手动编辑或通过界面修改。

## 使用教程

### 首次使用

1. 下载 [llama.cpp](https://github.com/ggml-org/llama.cpp/releases) 并解压
2. 确保 `llama-cli.exe` 和 `llama-server.exe` 在系统 PATH 中，或使用运行窗口的「llama.cpp 路径」选项指定可执行文件所在目录
3. 准备一个 GGUF 格式的模型文件
4. 启动 `llama_gui.exe`
5. 点击「📋 模型管理」→ 添加你的模型（名称 + 路径）
6. 回到主界面，点击「🚀 运行」→ 如 llama.cpp 未在 PATH 中，勾选「llama.cpp 路径」并浏览选择目录
7. 选择模型和参数，点击「📋 预览命令」检查生成的命令
8. 点击「▶ 确认运行」开始使用

### 命令行模式对话

1. 运行窗口选择「命令行」模式
2. 选择模型，调整参数（温度、重复惩罚等）
3. 勾选「多行输入模式」方便输入多行文本
4. 如需加速推理，勾选「MTP 推测解码」
5. 预览命令后执行，在 CMD 窗口中开始对话

### 网页模式

1. 运行窗口选择「网页」模式
2. 选择模型，设置端口（默认 8081）
3. 执行后自动打开浏览器访问 `http://127.0.0.1:端口`
4. 在浏览器中享受类似 ChatGPT 的对话体验

### 转换模型

1. 主界面点击「🔄 转化」
2. 填写 llama.cpp 路径（包含转换脚本的文件夹）
3. 填写 HF 模型源文件夹路径
4. 选择输出格式（如 f16 或 q8_0）
5. 选择输出文件保存位置
6. 点击「转换模型」并在确认框中检查命令
7. 在 CMD 窗口中查看转换进度

> 需要 Python 环境和 llama.cpp 自带的 `convert_hf_to_gguf.py` 脚本。

## 常见问题

### Q: 提示找不到 llama-cli 怎么办？

A: 确保 llama.cpp 的 `llama-cli.exe` 在系统 PATH 环境变量中，或者将 exe 复制到 `llama_gui.exe` 同目录下。可在 CMD 中运行 `llama-cli --version` 验证。

### Q: 如何添加新模型？

A: 点击主界面的「📋 模型管理」，输入模型名称，点击「浏览...」选择 .gguf 文件，点击「添加」即可。无需手动编辑配置文件。

### Q: 模型列表为空怎么办？

A: 首次使用需要手动添加模型。点击「模型管理」→ 输入名称和路径 → 添加。后续启动会自动加载已保存的列表。

### Q: 转换模型需要什么环境？

A: 需要安装 Python，且 llama.cpp 目录中包含 `convert_hf_to_gguf.py` 和 `convert_lora_to_gguf.py`。建议直接 `git clone https://github.com/ggml-org/llama.cpp` 获取完整仓库。

### Q: 配置文件在哪里？

A: 所有配置文件（`llama_config.txt`、`llama_models.txt`、`llama_convert_config.txt`）保存在 `llama_gui.exe` 同目录下。

### Q: 怎样使用 GPU 加速？

A: 增大 `ngl` 参数值（如 48 或 99），将更多模型层加载到 GPU。需要 CUDA 版本的 llama.cpp。

### Q: llama.cpp 路径选项怎么用？

A: 如果 `llama-cli.exe` 和 `llama-server.exe` 没有添加到系统 PATH 环境变量中，可以勾选「llama.cpp 路径」→ 点击「...」浏览选择 llama.cpp 的编译输出目录（如 `C:\llama.cpp\build\bin\Release`）→ 程序会自动使用完整路径调用可执行文件。首次设置后会自动保存，下次无需重新选择。

### Q: MTP 推测解码是什么？

A: Multi-Token Prediction，通过同时预测多个 token 来加速生成。适用于支持 MTP 的模型（如 Qwen 3.6 系列）。

### Q: 网页模式端口被占用怎么办？

A: 在运行窗口中修改端口号（如 8082、8083），确保端口未被其他程序占用。

### Q: 命令行模式下的额外参数怎么用？

A: 勾选「启用额外参数」，在输入框中填入任何 llama-cli 支持的参数，如：
```
--top-k 40 --top-p 0.9 --min-p 0.05 --chat-template llama3
```
点击「预览命令」可看到拼接后的完整命令。

### Q: 程序能记住上次的设置吗？

A: 是的。每次点击「预览命令」或「确认运行」时会自动保存当前参数到配置文件，下次启动自动恢复。

### Q: 点 X 关闭和点返回有什么区别？

A: **返回**按钮回到主界面继续操作；窗口右上角的 **X** 直接退出整个程序。

## 系统要求

### 运行要求

- Windows 7 / 8 / 10 / 11
- [llama.cpp](https://github.com/ggml-org/llama.cpp) 已安装（llama-cli.exe / llama-server.exe）
- 足够的 RAM（视模型大小而定，建议至少 8GB）
- （可选）NVIDIA GPU + CUDA 驱动，用于 GPU 加速推理

### 转换要求

- Python 3.8+
- llama.cpp 完整仓库（含转换脚本）
- 足够的磁盘空间存放转换后的 GGUF 文件

## v2 相比 v1 的改进

| 方面 | v1 | v2 |
|------|----|----|
| 代码组织 | 单文件 957 行 | 4 文件模块化，职责清晰 |
| 模型管理 | 手动编辑 txt 文件 | **图形化**增删改查，文件浏览 |
| 路径输入 | 纯手打 | **浏览按钮**一键选择 |
| 命令执行 | 一键运行，无法预览 | **预览 → 可编辑 → 确认运行** |
| 窗口导航 | 无返回机制 | **返回按钮**，X 退出程序 |
| 配置文件 | 相对路径 | 基于 exe 目录的**绝对路径** |
| MTP 支持 | 需手动输入参数 | **一键勾选** |
| UI 布局 | 控件堆叠 | GroupBox 分组，层次分明 |
| 错误处理 | 基本 | **输入验证 + 确认对话框** |
| 窗口注册 | 每次打开重新注册 | 启动时一次性注册 |
| 配置保存 | fclose 刷新 | **fflush + fclose** 确保落盘 |

## 开发说明

- 纯 C 语言编写，仅依赖 Windows API（无第三方库）
- 编译后为单文件 exe，约 46KB
- 代码使用 UTF-8 编码，注释为中文
- 使用模态子窗口消息循环模式管理多窗口

### 文件结构

```
llama_gui_v2/
├── common.h          # 共享头文件 — 类型、常量、接口
├── main.c            # 入口 + 主窗口 + 模型管理 + 工具函数
├── run_window.c      # 运行配置窗口
├── convert_window.c  # 模型转换窗口
├── llama_gui.exe     # 编译产物
└── README.md         # 本文件
```

### 编译

```bash
gcc -o llama_gui.exe main.c run_window.c convert_window.c \
    -mwindows -s -O2 -lole32 -lshell32 -lcomctl32
```
