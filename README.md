# Llama GUI 

一个简单易用的Llama模型启动和转换工具，支持Windows平台。
由AI主导编写。

## 功能特点

### 🚀 主启动界面
- 简洁的双按钮界面
-「运行」按钮：启动Llama运行界面
-「转化」按钮：启动模型转换界面

### 💻 运行界面
- **命令行模式**
  - 模型选择下拉框
  - 温度、重复惩罚、最大生成长度等参数
  - 多行输入模式开关
  - 自定义其他参数
  - 帮助按钮查看全部参数

- **网页模式**
  - 模型选择
  - 端口配置
  - 自动打开浏览器
  - 自动关闭设置界面

### 🔄 转换界面
- **转换模型**
  - 将HF格式转换为GGUF格式
  - 支持自定义outtype参数
  - 从配置文件读取路径

- **转换LoRA**
  - 将LoRA适配器转换为GGUF格式
  - 支持自定义outtype参数
  - 从配置文件读取路径

## 编译和安装

### 下载

- 从[Release页面](https://github.com/rhys1/llama_GUI/releases)下载最新版本的可执行文件。
- 或clone仓库到本地：
```bash
git clone https://github.com/rhys1/llama_GUI.git
```

### 编译要求
- Windows 7或更高版本
- MinGW GCC编译器

### 编译命令
```bash
gcc -o llama_gui.exe main.c -mwindows -s -O2
```

### 运行
直接双击生成的`llama_gui.exe`即可运行。

## 配置文件说明

### 主配置文件 - llama_config.txt
保存运行界面的参数设置，格式如下：
```
命令行/网页
模型名称
ctx-size值
ngl值
温度值
重复惩罚值
最大生成值
端口值
mli开关状态
其他参数开关状态
其他参数字符串
```

### 转换配置文件 - llama_convert_config.txt
保存转换界面的路径配置，格式如下：
```
# llama.cpp路径
llama.cpp路径在这里
# 模型文件路径
模型文件路径在这里
# lora文件路径
lora文件路径在这里
# 转换后模型存储路径
输出路径在这里
# outtype参数
f16
```

### 模型列表配置 - llama_models.txt
保存可选模型列表，格式如下：
```
模型数量
模型1名称
模型1完整路径
模型2名称
模型2完整路径
```

## 使用说明

### 首次运行
1. 启动程序会显示主界面
2. 点击「转化」或「运行」都需要配置相关文件


### 使用「运行」功能
1. 点击「运行」打开运行界面
2. 选择命令行或网页模式
3. 选择模型（需要先编辑模型列表文件）
4. 修改其他参数（可选）
5. 点击「运行命令」执行
6. 程序会自动保存参数配置

### 使用「转化」功能
1. 配置好`llama_convert_config.txt`中的所有路径
2. 点击「转化」打开转换界面
3. 修改outtype参数（可选）
4. 点击「转换模型」或「转换lora」执行
5. 转换结果会在新的CMD窗口中显示

## 参数说明

### 常用参数
- `--ctx-size`: 上下文窗口大小
- `-ngl`: GPU层数
- `--temp`: 温度参数
- `--repeat-penalty`: 重复惩罚
- `-n`: 最大生成长度
- `-mli`: 多行输入模式

### 转换参数
- `--outtype`: 输出类型（f16、f32、q8_0等）

## 系统要求

### 运行要求
- 下载并解压对应的Llama.cpp版本（使用英伟达GPU注意需要安装CUDA驱动）（详见[llama.cpp 官方仓库](https://github.com/llama-cpp/llama.cpp)）
- 将llama-cli.exe添加到环境变量PATH中
- 相应的模型文件
- 足够的内存和磁盘空间

### 转换要求
- Python环境
- llama.cpp转换脚本（convert_hf_to_gguf.py和convert_lora_to_gguf.py）（直接克隆llama.cpp仓库即可）

## 常见问题

### Q: 如何添加新模型？
A: 编辑`llama_models.txt`文件，按照格式添加模型名称和完整路径，修改模型数量。

### Q: 提示找不到配置文件怎么办？
A: 首次运行会自动创建默认配置文件，按照提示编辑好相关路径即可。

### Q: 转换模型失败怎么办？
A: 检查`llama_convert_config.txt`中的路径是否正确，确保有Python环境和转换脚本。

## 注意事项

1. 确保所有路径配置正确
2. 首次使用请先配置好转换路径
3. 运行过程中不要关闭CMD窗口
4. 网页模式下确保端口没有被占用

## 开发说明

代码使用纯Windows API编写，无第三方依赖，方便移植和修改。
